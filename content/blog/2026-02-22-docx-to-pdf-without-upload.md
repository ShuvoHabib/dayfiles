## What to verify before you send the file

The final review should be short, but it should be disciplined. A good operator does not reread the whole file unless the task demands it. They check the few items most likely to break the workflow outcome.

1. Page layout matches the source DOCX.
2. Headers, footers, and signatures are intact.
3. The PDF version is the one actually shared downstream.

If those checks pass, the file is usually ready for the next handoff. If one fails, the problem is still caught early enough to fix without resetting the whole workflow.

## Common mistakes with DOCX to PDF

1. **Exporting before the DOCX is final.** Use PDF as the release format after internal editing is complete.
2. **Assuming every font will embed cleanly.** Check the rendered PDF on the target device before sending it out.
3. **Treating conversion as a formality instead of a review checkpoint.** Read the exported PDF once because layout issues appear there, not in the DOCX editor.

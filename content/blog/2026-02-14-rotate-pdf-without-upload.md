## What to verify before you send the file

The final review should be short, but it should be disciplined. A good operator does not reread the whole file unless the task demands it. They check the few items most likely to break the workflow outcome.

1. All targeted pages face the correct direction.
2. Page order stays unchanged.
3. The final packet is readable in the destination device.

If those checks pass, the file is usually ready for the next handoff. If one fails, the problem is still caught early enough to fix without resetting the whole workflow.

## Common mistakes with Rotate PDF

1. **Applying a global rotation to a mixed packet.** Target specific pages when only part of the file is misaligned.
2. **Using orientation fixes after several other edits.** Correct page direction early so later reviews happen on the right view.
3. **Skipping a page-by-page scan of thumbnails.** Review thumbnails before export to catch one-off rotation errors.

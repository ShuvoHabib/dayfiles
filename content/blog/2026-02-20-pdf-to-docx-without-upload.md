## What to verify before you send the file

The final review should be short, but it should be disciplined. A good operator does not reread the whole file unless the task demands it. They check the few items most likely to break the workflow outcome.

1. Key headings and paragraphs survived conversion.
2. Tables and lists remain usable.
3. The edited DOCX is reviewed before re-export to PDF.

If those checks pass, the file is usually ready for the next handoff. If one fails, the problem is still caught early enough to fix without resetting the whole workflow.

## Common mistakes with PDF to DOCX

1. **Treating the converted DOCX as final without comparison.** Use the original PDF as the reference while you review formatting and missing elements.
2. **Converting a scan when OCR quality is uncertain.** Check whether the source is text-based or image-based before expecting clean edits.
3. **Editing first and planning quality control later.** Set a review pass before the document re-enters the delivery workflow.
